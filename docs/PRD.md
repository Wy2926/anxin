# 「安心」守护家长 App —— 产品需求文档（PRD v1.0）

> v1.0 在 v0.1 基础上补充：成功指标、数据模型、关键状态机、告警升级与通知策略、知情同意与审计模型、埋点、竞品分析、各里程碑验收标准、降级与离线行为。结构：1–8 为概览与功能；9–17 为工程化细化。

> 一款让子女远程守护、关怀、并轻度监督父母日常健康与安全的家庭关怀 App。
> 核心理念：**关怀 > 监控**。所有"看屏 / 定位 / 用药"等敏感能力必须建立在父母**知情、可控、可随时拒绝**的前提上。

---

## 1. 产品概述

### 1.1 一句话定位
帮助子女与年迈父母保持"看得见、联系得上、放得下心"的家庭守护工具。

### 1.2 背景与机会
- 中国 60 岁以上人口已超 2.9 亿，空巢/独居老人比例持续上升。
- 子女与父母常异地，信息断层导致：父母身体异常发现晚、用药不规律、易受电信诈骗、不会用智能手机求助。
- 现有产品要么偏"医疗硬件"（手环/血压计）、要么偏"通讯"（微信视频），缺少一个**以子女为运营核心、以父母为被守护对象**的轻量统一入口。

### 1.3 目标用户
| 角色 | 画像 | 核心诉求 |
|---|---|---|
| **子女（守护者 Guardian）** | 25–50 岁，异地工作，1–4 位长辈需关注 | 随时知道父母状态、能远程帮忙、出事能第一时间响应 |
| **父母（被守护者 Elder）** | 55–85 岁，智能机使用能力弱，视力/听力下降 | 操作简单、不被打扰、保留尊严、紧急时一键求助 |
| **其他亲属（协同守护者）** | 兄弟姐妹、配偶 | 多人协同、信息共享、责任分担 |

### 1.4 设计原则
1. **尊严优先**：父母端永远知道"谁在看我、看了什么"，可一键关闭任何监督项。
2. **极简适老**：父母端大字、大按钮、强对比、少层级、可语音操作。
3. **知情同意 + 可审计**：所有敏感访问留痕，双方可查。
4. **轻打扰**：默认"被动守护"（异常才提醒），而非"持续监视"。
5. **可降级**：网络/设备差也能用最基础的"打电话 + 报平安"。

## 2. 角色与权限模型

### 2.1 双端 + 一云
- **子女端 App（Guardian App）**：功能完整，运营守护关系。
- **父母端 App（Elder App）**：极简界面，可由子女远程"编排"。
- **云端服务**：账号、家庭关系、消息、媒体中转、提醒调度、告警。

### 2.2 家庭与权限
- 一个"家庭组（Family）"包含 1 位被守护者 + N 位守护者。
- 守护权限分级，**被守护者拥有最终否决权**：

| 权限项 | 默认 | 说明 |
|---|---|---|
| 查看在线状态/电量/步数 | 开 | 低敏感 |
| 语音 / 视频通话 | 需接听 | 父母主动接听才接通 |
| 远程协助看屏（屏幕共享） | **关，单次授权** | 每次发起需父母点"同意"，限时，结束自动断开 |
| 实时定位 | 关 | 需父母一次性或长期授权 |
| 用药/日程提醒管理 | 开（可关） | 子女代为设置，父母可见可改 |
| 紧急 SOS 联系人 | 开 | 父母端常驻 |

### 2.3 隐私护栏（硬约束）
- "看屏"采用**邀请-同意制**：子女发起 → 父母端弹窗大按钮"同意/拒绝" → 同意后限时共享（如 10 分钟）→ 顶部常驻"正在被协助"红条 → 父母可随时一键结束。
- 严禁"静默/后台偷看"。任何监督项都有父母端可见的状态指示与操作日志。

## 3. 功能需求（按模块）

### M1. 账号与家庭组
- 手机号/微信登录；子女创建家庭组，生成邀请码/二维码。
- 子女可代父母完成注册与设备初始化（亲情代办）。
- 多守护者加入、角色设置（主守护人/协同）。

### M2. 实时通讯
- **语音通话 / 视频通话**（1:1 与家庭多人）。
- **一键大字视频**：父母端首屏直接显示子女头像，点头像即拨。
- **留言/语音条**：父母不便接听时留言。
- 通话质量自适应、弱网降级为语音。

### M3. 远程协助看屏（屏幕共享）
- 子女发起协助 → 父母同意 → 实时看到父母手机屏幕。
- 子女可"圈一圈/指一指"（标注）引导父母操作；可选远程操作需更高一级单独授权。
- 全程红条提示 + 结束即断 + 日志留痕。
- 典型场景：教父母用微信、帮父母辨别诈骗短信、协助挂号缴费。

### M4. 用药守护
- 子女/父母录入药品（名称、剂量、频次、餐前餐后、库存）。
- 定时提醒：父母端大字+语音+铃声提醒；可拍照识别药盒辅助录入。
- **服药确认闭环**：父母点"已服用" → 子女端看到打卡；超时未确认 → 升级提醒子女。
- 用药日历与依从率统计；余药不足提醒补药。

### M5. 健康与安全守护
- **报平安**：父母端一键"我很好"；或基于活动自动判断（手机长时间无活动 → 提醒子女）。
- **SOS 紧急求助**：父母端大红按钮，触发后向所有守护者推送（含定位、自动拨号、可录音）。
- **跌倒/异常检测（可选，依赖传感器/穿戴）**：作为增强项。
- 健康数据记录：血压、血糖、心率（手动录入或接入硬件），趋势图，异常提醒。
- **定位与安全区**：可选实时定位、电子围栏（走出常住区域提醒）。

### M6. 日程与生活关怀
- 子女为父母编排日程：吃药、复诊、量血压、生日提醒、缴费。
- 天气/穿衣提醒、节日关怀、亲情打卡（每日问候）。

### M7. 远程编排父母端界面（特色能力）✅ 原型已落地
- 子女在自己 App 中**可视化配置父母端首屏**：
  - 选择/排序快捷入口（打电话、视频、吃药、相册、收音机、遇事问子女；SOS 常驻锁定）。
  - 调整字号（1.0–1.4×）、高对比模式、是否开启语音播报。
  - 一键推送到父母设备生效（父母端弹出「孩子帮你调整了主屏」提示，可还原）。
- 预置"模板"：极简版、标准版、丰富版。
- **原型实现**：子女端「我的」→「编排父母端手机界面」进入全屏编排页，含**父母端实时预览缩略机**；点「推送到设备」→ 父母端首屏即时重排、应用字号/对比度/语音，并在消息中心留痕。
  代码：`src/guardian/Orchestrator.tsx`、`src/elder/ElderApp.tsx`（按 `elderLayout` 渲染）、`src/store.tsx`（`ElderLayout` + `APPLY_ELDER_LAYOUT`）。

### M8. 防诈骗与协助
- 可疑短信/来电提醒、陌生链接拦截提示。
- "遇事问子女"按钮：一键把当前页面/短信转给子女判断。

### M9. 消息与告警中心
- 子女端聚合：服药打卡、SOS、异常、设备离线、低电量等。
- 分级（普通/重要/紧急），紧急可强提醒（突破勿扰）。

## 4. 关键用户故事（节选）
- 作为子女，我希望父母超时没吃降压药时收到提醒，以便打电话催。
- 作为子女，我希望能远程看父母的手机屏幕，手把手教他用医院挂号小程序。
- 作为父母，我希望界面只有"给儿子打电话"和"求救"两个大按钮，别的别来烦我。
- 作为父母，我希望在被子女看屏时清楚知道"正在被看"，并能随时关掉。
- 作为协同守护的女儿，我希望和哥哥共享妈妈的健康记录，不用重复问。

## 5. 非功能需求
- **适老化合规**：参照《移动互联网应用（App）适老化通用设计规范》（大字、语音、防误触、简化流程、去广告弹窗）。
- **隐私合规**：符合《个人信息保护法》《数据安全法》；敏感权限最小化、单独同意、可撤回、可导出/删除；数据加密（传输 TLS、敏感字段存储加密）。
- **可靠性**：SOS/告警链路高可用，多通道送达（推送+短信+电话）。
- **性能**：父母端低端机可用、低流量模式；视频弱网降级。
- **可访问性**：支持系统读屏、超大字号、色盲友好配色。

## 6. 范围与里程碑（建议）
- **MVP（P0）**：家庭组、语音/视频、用药提醒+服药确认、SOS、报平安、父母端极简首屏、消息中心。
- **P1**：远程看屏协助、远程编排父母端界面、定位/安全区、健康数据趋势。
- **P2**：防诈骗、穿戴/血压计硬件接入、跌倒检测、多语言/方言语音。

## 7. 技术与平台（待与用户确认，见 docs/OPEN_QUESTIONS.md）
- 平台：iOS + Android 双端（父母多为 Android）。建议跨端框架（Flutter / React Native）+ 原生模块（音视频、屏幕共享、后台保活）。
- 音视频/屏幕共享：声网 Agora / 腾讯 TRTC 等 RTC 方案（含屏幕采集）。
- 后端：账号、关系、提醒调度、推送、媒体信令；推送多通道（厂商推送 + 短信）。

## 8. 风险
- **屏幕共享/定位的隐私与合规风险** → 严格同意制 + 审计 + 默认关闭。
- **后台保活与电量**（Android 厂商限制）→ 引导加白名单 + 多通道告警兜底。
- **老人接受度**：抵触"被监控" → 文案与交互强调"关怀/协助"，父母完全可控。
- **SOS 误触/漏报** → 防误触 + 二次确认（紧急时可跳过）+ 多通道。

## 9. 成功指标（North Star 与 KPI）

**North Star：每周「被有效守护的家长数」** = 至少满足以下任一的活跃家长占比 ≥ 阈值：当周有 ≥1 次双向通话 / 用药打卡 / 报平安。它同时反映"子女在用"与"父母被覆盖"。

| 维度 | 指标 | 目标（上线 6 个月） |
|---|---|---|
| 激活 | 家庭组配对完成率（创建组→父母端激活） | ≥ 70% |
| 激活 | 父母端首日完成 1 次关键操作（接听/报平安/打卡） | ≥ 60% |
| 留存 | 子女端次月留存 | ≥ 55% |
| 留存 | 父母端周活（被动守护也算活跃） | ≥ 75% |
| 用药 | 7 日服药依从率（已打卡 / 应服用） | ≥ 80% |
| 用药 | 超时漏服的升级触达成功率 | ≥ 95% |
| 安全 | SOS 端到端送达时延（触发→子女收到强提醒） | P95 ≤ 10s |
| 安全 | SOS 至少 1 位守护者"已响应"率 | ≥ 99% |
| 通讯 | 一次拨通率（首次呼叫即接通） | ≥ 85% |
| 信任 | 父母端"敏感访问"投诉 / 误用率 | ≈ 0 |
| 商业 | 家庭付费转化（若启用订阅） | 行业基准对齐 |

> 反向护栏指标：父母端"被打扰频次"（弹窗/提醒次数/日）需设上限，过高即降权。

## 10. 核心数据模型（实体与关系）

```
User ──< GuardianMembership >── Family ──1:1── Elder(Profile)
Family ──< Device(Elder/Guardian) 
Elder  ──< Medication ──< MedEvent(打卡/漏服)
Elder  ──< HealthReading(bp/glucose/hr/steps)
Elder  ──< Reminder(用药/复诊/缴费/生日)
Elder  ──< ElderLayout(首屏编排，含版本)
Family ──< Alert(info/warn/urgent)
Family ──< AuditLog(敏感访问留痕)
Family ──< AssistSession(看屏会话)
Family ──< Consent(权限授权记录)
```

| 实体 | 关键字段 | 说明 |
|---|---|---|
| `Family` | id, ownerGuardianId, createdAt | 一被守护者 + N 守护者 |
| `GuardianMembership` | userId, familyId, role(`owner`/`co`), perms | 角色与权限 |
| `Elder` | id, name, relation, online, battery, steps, heart, lastSafeReportAt | 被守护者状态 |
| `Medication` | id, name, dose, time, note(餐前/后), stock, scheduleRRULE | 用药计划 |
| `MedEvent` | medId, dueAt, status(`taken`/`snoozed`/`missed`), actedAt, actor | 闭环事件，统计依从率 |
| `HealthReading` | type, value, takenAt, source(`manual`/`device`) | 趋势与异常判断 |
| `ElderLayout` | template, fontScale, highContrast, voice, tiles[], version, pushedAt | 远程编排（M7） |
| `Alert` | level, title, detail, elderId, createdAt, ackBy[] | 消息/告警 |
| `AssistSession` | startedAt, endedAt, grantedBy, durationLimit, annotations | 看屏会话（强同意+限时） |
| `Consent` | scope(`screen`/`location`/…), grant(`once`/`always`/`off`), grantedAt, revokedAt | 同意记录，可撤回 |
| `AuditLog` | actor, action, scope, elderId, at | 不可篡改追加日志，双方可查 |

> 原型映射：`src/store.tsx` 已实现 `Elder / Medication / Alert / ElderLayout / ScreenAssist`。v1.0 在此模型上补 `HealthReading`、`AuditLog`、`AssistSession`、`Consent`。

## 11. 关键状态机

### 11.1 用药闭环 + 升级（MedEvent）
```
scheduled ──到点──> due ──父母「已吃」──> taken
   │                  │
   │                  ├─「稍后」──> snoozed(+10min) ──> due (最多 N 次)
   │                  └─超时T1未操作──> 父母端再响 + 子女端 warn
                                          └─超时T2仍无──> missed
                                              → 子女强提醒(推送+短信) + 可一键呼叫
```
参数：`T1=10min, T2=30min, 最大 snooze=2 次`（可配）。`missed` 计入依从率分母。

### 11.2 SOS 多通道升级
```
按住/连点 🆘 ──> 倒计时 3s（可取消）──> fired
  fired → 并行：① App 强推送(突破勿扰) 给全部守护者
              ② 5s 内无人「已响应」→ 自动短信
              ③ 15s 内仍无 → 依优先级自动外呼守护者
              ④ 附带：实时定位、最近状态、可选现场录音
  任一守护者点「已响应」→ 其余通道降级为通知；记录响应人与时延
```
兜底：父母端可设"紧急联系人/社区/120"为末位通道。

### 11.3 看屏会话（强同意 + 限时 + 留痕）
```
guardian: requestAssist ──> elder: 弹窗[同意/拒绝]
  拒绝 → idle，写 AuditLog(declined)
  同意 → active：父母端顶部红条常驻；限时(默认10min)倒计时
        guardian 可标注；如需"远程操作"→ 二次单独授权
  结束条件(任一)：父母点结束 / 限时到 / guardian 主动结束 / 断网
        → 立即断开，写 AuditLog(session: 起止/时长/是否标注/是否远程操作)
```

## 12. 告警分级与通知策略

| 级别 | 示例 | 触达通道 | 勿扰 |
|---|---|---|---|
| info | 报平安、服药打卡、布局已推送 | App 内消息 + 角标 | 遵守勿扰 |
| warn | 用药超时 T1、低电量、设备离线、余药不足、健康读数偏高 | App 推送 | 遵守勿扰（可设例外） |
| urgent | SOS、用药 missed、走出安全区、跌倒 | 推送 + 短信 + 外呼，**突破勿扰** | 强制 |

策略要点：
- **去重与聚合**：同类 info 24h 内聚合为一条，避免轰炸父母与子女。
- **多守护者路由**：urgent 广播全部；warn 默认仅主守护人，可设"同时通知"。
- **静默时段**：父母端非紧急提醒默认 22:00–07:00 静音（用药等关键提醒可白名单）。
- **回执**：urgent 必须有人 `ack`，否则持续升级并提示其他守护者。

## 13. 知情同意与隐私/审计模型（硬约束细化）

- **最小必要 + 单独同意**：定位、看屏、录音、健康数据各自独立授权，默认关闭；可"单次/长期/关闭"三态，长期授权父母端常驻可见且可一键撤回。
- **可见性**：任何敏感能力开启期间，父母端有持续状态指示（红条/角标/语音播报），杜绝静默后台。
- **审计**：所有敏感访问（看屏、定位、读取健康、推送布局）写入 `AuditLog`，**双方可查、不可篡改、可导出**。
- **数据生命周期**：录音/屏幕内容默认不留存（仅实时流）；如需留存须再次单独同意并设保留期（默认 7 天自动删除）。
- **权利**：父母（或其代理）可查看/导出/删除个人数据，可解除某守护者权限或退出家庭组。
- 原型对应：`src/guardian/GuardianApp.tsx` 的「权限与隐私」开关 + 「隐私护栏」卡片；v1.0 新增「隐私 / 审计日志」视图展示 `AuditLog`。

## 14. 埋点与分析事件（节选）

| 事件 | 关键属性 | 用途 |
|---|---|---|
| `family_pair_complete` | familyId, elapsedSec | 激活漏斗 |
| `call_start / call_connect / call_end` | type, durationSec, network | 一次拨通率、弱网降级 |
| `med_due / med_taken / med_snoozed / med_missed` | medId, latencySec | 依从率、升级有效性 |
| `sos_fired / sos_acked` | channel, ackLatencySec, ackBy | 安全链路 SLA |
| `assist_request / assist_accept / assist_decline / assist_end` | durationSec, annotated | 看屏使用与信任 |
| `layout_pushed` | template, tileCount, fontScale | M7 采用度 |
| `safe_report` | source(manual/auto) | 被动守护活跃 |

隐私：埋点不含屏幕/录音内容；敏感字段脱敏；遵守可撤回与不追踪未授权数据。

## 15. 离线与降级行为

- **网络弱/断**：视频→语音→留言逐级降级；看屏会话断网即结束并留痕。
- **父母端极限降级**：无网络也能本地"打电话（运营商电话）"与显示紧急联系人，SOS 退化为直接拨打预设号码 + 发送本地短信。
- **后台保活（Android）**：引导加入厂商白名单；告警走厂商推送 + 短信双通道兜底，不依赖 App 在前台。
- **低端机/低流量模式**：关闭动画、降码率、按需加载，父母端首屏纯静态可秒开。

## 16. 竞品与差异化

| 方向 | 代表 | 局限 | 安心的差异 |
|---|---|---|---|
| 通讯 | 微信视频 | 无守护语义、无用药/SOS、父母操作门槛高 | 以"守护关系"为核心，父母端极简、子女可远程编排 |
| 医疗硬件 | 手环/血压计 App | 偏数据、缺关怀与协助、各自为政 | 软件统一入口，可选接入硬件作为增强 |
| 定位看护 | 儿童/老人定位器 | 偏"监控"、隐私争议、父母抵触 | **强同意 + 可审计 + 默认关闭**，关怀叙事 |
| 适老桌面 | 老人桌面启动器 | 单机、子女不可远程协同 | 双端联动，子女远程编排父母首屏（M7 特色） |

**护城河**：双端联动 + 远程编排 + 强同意隐私模型构成的"被信任的家庭守护网络"。

## 17. 里程碑与验收标准

### P0 / MVP — 验收
- 家庭组：子女创建组并生成邀请码；父母端扫码/输码 5 分钟内完成激活（可子女代办）。
- 通讯：1:1 语音/视频可拨通；弱网自动降级为语音；可留言。
- 用药闭环：到点全屏大字+语音提醒；「已吃」→ 子女端打卡且余药-1；超时按 11.1 升级到子女。
- 安全：SOS 满足 11.2（3s 可取消、多通道、必须有人 ack）；「我很好」报平安实时同步。
- 父母端极简首屏 + 子女远程编排（M7）：切模板/字号/对比/语音/排序，**推送后父母端即时重排**并留痕。
- 消息中心：聚合 info/warn/urgent，urgent 突破勿扰。
- 隐私：敏感能力默认关、单独同意、父母端状态可见、写 AuditLog。

### P1 — 验收
- 远程看屏满足 11.3（强同意+限时+红条+留痕+可标注）。
- 健康趋势：手动/设备录入 BP/血糖/心率，7/30 日趋势与异常提醒。
- 定位/安全区：单独授权、电子围栏越界 warn。

### P2 — 验收
- 防诈骗提示与「遇事问子女」转交闭环。
- 硬件接入（血压计/手环）与跌倒检测增强。
- 多语言/方言语音播报与指令。
