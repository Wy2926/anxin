# 「安心」守护家长 App —— 产品需求文档（PRD v0.1）

> 一款让子女远程守护、关怀、并轻度监督父母日常健康与安全的家庭关怀 App。
> 核心理念：**关怀 > 监控**。所有"看屏 / 定位 / 用药"等敏感能力必须建立在父母**知情、可控、可随时拒绝**的前提上。

---

## 1. 产品概述

### 1.1 一句话定位
帮助子女与年迈父母保持"看得见、联系得上、放得下心"的家庭守护工具。

### 1.2 背景与机会
- 中国 60 岁以上人口已超 2.9 亿，空巢/独居老人比例持续上升。
- 子女与父母常异地，信息断层导致：父母身体异常发现晚、用药不规律、易受电信诈骗、不会用智能手机求助。
- 现有产品要么偏"医疗硬件"（手环/血压计）、要么偏"通讯"（微信视频），缺少一个**以子女为运营核心、以父母为被守护对象**的轻量统一入口。

### 1.3 目标用户
| 角色 | 画像 | 核心诉求 |
|---|---|---|
| **子女（守护者 Guardian）** | 25–50 岁，异地工作，1–4 位长辈需关注 | 随时知道父母状态、能远程帮忙、出事能第一时间响应 |
| **父母（被守护者 Elder）** | 55–85 岁，智能机使用能力弱，视力/听力下降 | 操作简单、不被打扰、保留尊严、紧急时一键求助 |
| **其他亲属（协同守护者）** | 兄弟姐妹、配偶 | 多人协同、信息共享、责任分担 |

### 1.4 设计原则
1. **尊严优先**：父母端永远知道"谁在看我、看了什么"，可一键关闭任何监督项。
2. **极简适老**：父母端大字、大按钮、强对比、少层级、可语音操作。
3. **知情同意 + 可审计**：所有敏感访问留痕，双方可查。
4. **轻打扰**：默认"被动守护"（异常才提醒），而非"持续监视"。
5. **可降级**：网络/设备差也能用最基础的"打电话 + 报平安"。

## 2. 角色与权限模型

### 2.1 双端 + 一云
- **子女端 App（Guardian App）**：功能完整，运营守护关系。
- **父母端 App（Elder App）**：极简界面，可由子女远程"编排"。
- **云端服务**：账号、家庭关系、消息、媒体中转、提醒调度、告警。

### 2.2 家庭与权限
- 一个"家庭组（Family）"包含 1 位被守护者 + N 位守护者。
- 守护权限分级，**被守护者拥有最终否决权**：

| 权限项 | 默认 | 说明 |
|---|---|---|
| 查看在线状态/电量/步数 | 开 | 低敏感 |
| 语音 / 视频通话 | 需接听 | 父母主动接听才接通 |
| 远程协助看屏（屏幕共享） | **关，单次授权** | 每次发起需父母点"同意"，限时，结束自动断开 |
| 实时定位 | 关 | 需父母一次性或长期授权 |
| 用药/日程提醒管理 | 开（可关） | 子女代为设置，父母可见可改 |
| 紧急 SOS 联系人 | 开 | 父母端常驻 |

### 2.3 隐私护栏（硬约束）
- "看屏"采用**邀请-同意制**：子女发起 → 父母端弹窗大按钮"同意/拒绝" → 同意后限时共享（如 10 分钟）→ 顶部常驻"正在被协助"红条 → 父母可随时一键结束。
- 严禁"静默/后台偷看"。任何监督项都有父母端可见的状态指示与操作日志。

## 3. 功能需求（按模块）

### M1. 账号与家庭组
- 手机号/微信登录；子女创建家庭组，生成邀请码/二维码。
- 子女可代父母完成注册与设备初始化（亲情代办）。
- 多守护者加入、角色设置（主守护人/协同）。

### M2. 实时通讯
- **语音通话 / 视频通话**（1:1 与家庭多人）。
- **一键大字视频**：父母端首屏直接显示子女头像，点头像即拨。
- **留言/语音条**：父母不便接听时留言。
- 通话质量自适应、弱网降级为语音。

### M3. 远程协助看屏（屏幕共享）
- 子女发起协助 → 父母同意 → 实时看到父母手机屏幕。
- 子女可"圈一圈/指一指"（标注）引导父母操作；可选远程操作需更高一级单独授权。
- 全程红条提示 + 结束即断 + 日志留痕。
- 典型场景：教父母用微信、帮父母辨别诈骗短信、协助挂号缴费。

### M4. 用药守护
- 子女/父母录入药品（名称、剂量、频次、餐前餐后、库存）。
- 定时提醒：父母端大字+语音+铃声提醒；可拍照识别药盒辅助录入。
- **服药确认闭环**：父母点"已服用" → 子女端看到打卡；超时未确认 → 升级提醒子女。
- 用药日历与依从率统计；余药不足提醒补药。

### M5. 健康与安全守护
- **报平安**：父母端一键"我很好"；或基于活动自动判断（手机长时间无活动 → 提醒子女）。
- **SOS 紧急求助**：父母端大红按钮，触发后向所有守护者推送（含定位、自动拨号、可录音）。
- **跌倒/异常检测（可选，依赖传感器/穿戴）**：作为增强项。
- 健康数据记录：血压、血糖、心率（手动录入或接入硬件），趋势图，异常提醒。
- **定位与安全区**：可选实时定位、电子围栏（走出常住区域提醒）。

### M6. 日程与生活关怀
- 子女为父母编排日程：吃药、复诊、量血压、生日提醒、缴费。
- 天气/穿衣提醒、节日关怀、亲情打卡（每日问候）。

### M7. 远程编排父母端界面（特色能力）
- 子女在自己 App 中**可视化配置父母端首屏**：
  - 选择/排序快捷入口（拨打某子女、SOS、吃药、收音机、相册）。
  - 调整字号、对比度、是否开启语音播报。
  - 一键推送到父母设备生效（父母端可见变更，可还原）。
- 预置"模板"：极简版（只有打电话+SOS）、标准版、丰富版。

### M8. 防诈骗与协助
- 可疑短信/来电提醒、陌生链接拦截提示。
- "遇事问子女"按钮：一键把当前页面/短信转给子女判断。

### M9. 消息与告警中心
- 子女端聚合：服药打卡、SOS、异常、设备离线、低电量等。
- 分级（普通/重要/紧急），紧急可强提醒（突破勿扰）。

## 4. 关键用户故事（节选）
- 作为子女，我希望父母超时没吃降压药时收到提醒，以便打电话催。
- 作为子女，我希望能远程看父母的手机屏幕，手把手教他用医院挂号小程序。
- 作为父母，我希望界面只有"给儿子打电话"和"求救"两个大按钮，别的别来烦我。
- 作为父母，我希望在被子女看屏时清楚知道"正在被看"，并能随时关掉。
- 作为协同守护的女儿，我希望和哥哥共享妈妈的健康记录，不用重复问。

## 5. 非功能需求
- **适老化合规**：参照《移动互联网应用（App）适老化通用设计规范》（大字、语音、防误触、简化流程、去广告弹窗）。
- **隐私合规**：符合《个人信息保护法》《数据安全法》；敏感权限最小化、单独同意、可撤回、可导出/删除；数据加密（传输 TLS、敏感字段存储加密）。
- **可靠性**：SOS/告警链路高可用，多通道送达（推送+短信+电话）。
- **性能**：父母端低端机可用、低流量模式；视频弱网降级。
- **可访问性**：支持系统读屏、超大字号、色盲友好配色。

## 6. 范围与里程碑（建议）
- **MVP（P0）**：家庭组、语音/视频、用药提醒+服药确认、SOS、报平安、父母端极简首屏、消息中心。
- **P1**：远程看屏协助、远程编排父母端界面、定位/安全区、健康数据趋势。
- **P2**：防诈骗、穿戴/血压计硬件接入、跌倒检测、多语言/方言语音。

## 7. 技术与平台（待与用户确认，见 docs/OPEN_QUESTIONS.md）
- 平台：iOS + Android 双端（父母多为 Android）。建议跨端框架（Flutter / React Native）+ 原生模块（音视频、屏幕共享、后台保活）。
- 音视频/屏幕共享：声网 Agora / 腾讯 TRTC 等 RTC 方案（含屏幕采集）。
- 后端：账号、关系、提醒调度、推送、媒体信令；推送多通道（厂商推送 + 短信）。

## 8. 风险
- **屏幕共享/定位的隐私与合规风险** → 严格同意制 + 审计 + 默认关闭。
- **后台保活与电量**（Android 厂商限制）→ 引导加白名单 + 多通道告警兜底。
- **老人接受度**：抵触"被监控" → 文案与交互强调"关怀/协助"，父母完全可控。
- **SOS 误触/漏报** → 防误触 + 二次确认（紧急时可跳过）+ 多通道。
