# 「安心」UI/UX 设计方案 v1.0

本文件描述两端（子女端 Guardian / 父母端 Elder）的信息架构、关键流程、界面布局（线框）、视觉与适老化规范。线框采用 ASCII 示意，便于后续转 Figma。

---

## 1. 设计目标与基调
- **子女端**：信息密度适中、效率优先、状态一目了然（"父母今天怎么样"一眼看清）。
- **父母端**：极简、大字、大按钮、强反馈、可语音；尽量"一屏一事"。
- 情感基调：温暖、安心、可信赖（非冷硬的"监控系统"感）。

## 3. 关键用户流程

### 3.1 首次配对（子女主导）
```
子女: 注册 → 创建家庭组 → 生成邀请二维码
          ↓
父母: 安装父母端 → 扫码/输码 → 一键加入（子女可代填）
          ↓
子女: 远程编排父母首屏 → 设置用药/SOS联系人 → 完成
```

### 3.2 用药提醒闭环
```
[到点] 父母端: 全屏大字+语音"该吃降压药啦" + [已吃] [稍后]
   ├ 点"已吃"  → 子女端打卡✓ + 余药-1
   ├ 点"稍后"  → 10分钟后再提醒
   └ 超时未操作 → 升级：再响 → 仍无 → 推送+短信通知子女
```

### 3.3 远程看屏协助（强同意）
```
子女端: [发起协助] → 父母端弹窗(大按钮): "孩子想帮你看看手机，同意吗？"
                                  [同意]        [拒绝]
   同意 → 顶部红条"正在被协助 ●  [结束]" 全程常驻
        → 子女看到屏幕，可标注圈画引导
        → 限时10分钟/父母点结束 → 立即断开 → 双方留日志
```

### 3.4 SOS 求助
```
父母: 长按/连点 🆘 (防误触) → 倒计时3s可取消 → 触发
   → 向所有守护者: 强推送 + 自动短信 + 依次拨号
   → 附带: 实时定位、最近状态、可选现场录音
   → 子女端: 全屏红色告警，一键回拨/导航/呼叫120
```

## 6. 视觉规范（草案）

### 6.1 色彩
- 主色：温暖青绿 `#1F9E8C`（安心、健康）。
- 辅助：暖橙 `#F2A14E`（关怀提示）。
- 状态：正常 `#2BB673`、注意 `#F2A14E`、紧急 `#E5484D`。
- 父母端：高对比，背景近白 `#FFFFFF`，文字近黑 `#1A1A1A`，避免低对比灰字。

### 6.2 字体与尺寸
- 子女端正文 14–16pt；父母端正文 ≥ 20pt、按钮文字 ≥ 24pt、关键操作 ≥ 28pt。
- 行高宽松（1.5×），数字与重要词加粗。

### 6.3 触控与反馈
- 父母端按钮最小点击区 ≥ 64×64dp，间距大、防误触。
- 每次操作有视觉+震动+语音三重反馈。
- 错误/危险操作有二次确认（紧急 SOS 除外，倒计时取消即可）。

### 6.4 可访问性
- 支持系统超大字体、读屏（TalkBack/VoiceOver）。
- 色盲友好：状态不仅靠颜色，辅以图标/文字。
- 全局语音指令入口（"打电话给儿子""我要吃的药"）。

## 7. 文案语气
- 父母端：口语、亲切、短句。例：「该吃降压药啦」「点这个绿色按钮给儿子打电话」。
- 涉及监督的能力一律以"关怀/帮忙"表述，并明确告知父母可控。例：「孩子想帮你看看手机」而非「远程监控」。

## 8. 交付物路线（设计）
1. 本线框 →（与用户确认后）Figma 低保真 → 高保真 → 交互原型。
2. 组件库：按钮、卡片、提醒弹窗、状态徽标、红条、SOS。
3. 父母端"模板"三套可视化预览。

## 9. 设计 Token（与原型 `src/styles.css` 对齐）

### 9.1 色彩
| Token | 值 | 用途 |
|---|---|---|
| `--brand` / `--brand-2` / `--brand-ink` | `#10b3a3` / `#0e9aa0` / `#0a6b66` | 品牌青绿（安心/健康） |
| `--accent` | `#6c5ce7` | 强调紫（相册/编排） |
| `--warm` | `#ff8f3c` | 关怀暖橙（用药提醒） |
| `--ok` / `--warn` / `--urgent` | `#16b364` / `#f5a623` / `#f0455a` | 正常 / 注意 / 紧急 |
| `--ink` / `--ink-2` / `--muted` | `#0d1620` / `#3b4a59` / `#8a99a8` | 文字三级 |
| `--bg` / `--card` / `--line` | `#f3f5f8` / `#fff` / `#eef2f5` | 背景 / 卡面 / 分隔 |

状态语义**不只靠颜色**：正常🟢/注意🟠/紧急🔴 同时配图标与文字（色盲友好）。

### 9.2 圆角 / 阴影 / 间距
- 圆角：`--r-sm 12 / --r-md 18 / --r-lg 26`。
- 阴影：`--shadow-card`（层次卡）、`--shadow-pop`（弹层）。
- 间距基数 4px；卡片内边距 18，元素间距 10–14。

### 9.3 字号
| | 子女端 | 父母端（基准，再乘 `fontScale 1.0–1.4×`） |
|---|---|---|
| 正文 | 14–16 | ≥ 20 |
| 次要 | 12–13 | ≥ 16 |
| 标题/数字 | 18–23 | ≥ 24 |
| 关键按钮文字 | 14 | ≥ 24（SOS/已吃 ≥ 28） |

父母端字号由 `--e-scale` 统一缩放（`calc(基准 * var(--e-scale))`），子女远程编排即时生效。

## 13. 语音交互
- 父母端全局麦克风：识别"打电话给儿子""我要吃的药""我很好""救命"等意图直达对应动作。
- 关键提醒**语音播报**（可在编排中开关）：用药/看屏同意框朗读，听障可关。
- 文案口语化、单意图、短句；识别失败给出大按钮兜底，绝不让父母卡死。

## 14. 动效与反馈
- 时长：进入 200ms / 弹层 260ms（`cubic-bezier(.2,.8,.2,1)`）；尊重"减少动态效果"系统设置。
- 父母端每次操作三重反馈：视觉（按压/勾选）+ 震动 + 语音。
- "正在协助/求助"用脉冲红点（`pulse 1.4s`）持续提示，不可忽视。

## 15. 暗色 / 夜间
- 子女端支持系统深色：背景 `#0d1620` 系、卡面提亮、品牌色保持。
- 父母端默认高亮高对比；夜间仅降低背景亮度、保留大字与高对比，**不引入低对比灰字**。

## 16. 可访问性checklist（适老规范对齐）
- [ ] 支持系统超大字体与 App 内 1.0–1.4× 双重缩放
- [ ] 读屏（TalkBack/VoiceOver）全流程可用，控件有 `aria` 语义
- [ ] 触控目标 ≥ 64×64dp，间距防误触
- [ ] 对比度 ≥ WCAG AA（父母端关键文字 AAA）
- [ ] 状态不只靠颜色（图标+文字冗余）
- [ ] 无强制广告/弹窗，关键路径 ≤ 2 步
- [ ] 危险操作二次确认（SOS 例外：倒计时取消）
